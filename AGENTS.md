# Fullstack Boilerplate — Agent Guide

> **Last updated:** 2026-05-11 — DHI Dockerfiles, chi docgen, SEO meta, middleware refactor, dep updates

## Commands

```bash
# Backend
cd backend && go run ./cmd/server                   # start API on :8080
cd backend && go run ./cmd/server -routes           # print API routes (Markdown) and exit
cd backend && go test ./...                          # run all tests (none exist yet)
cd backend && bash scripts/seed.sh                   # seed admin@boilerplate.com / password123

# Frontend
cd frontend && npm run dev                           # dev server on :5173
cd frontend && npm run build                         # tsc --noEmit + vite build (lint → typecheck → build)
cd frontend && npm run lint                          # ESLint

# Docker
docker compose up -d postgres keydb                  # infra only (run backend locally)
docker compose up -d                                 # full stack
```

## Architecture

**Backend** — chi router, pgx pool (20 max / 5 min conns), Bun ORM, go-playground/validator, KeyDB/Redis.

ID generation uses `crypto/rand` directly (not `google/uuid`). Bcrypt cost 12; missing-user responses use a dummy hash to prevent email enumeration via timing.

Periodic cleanup goroutine in `main.go` purges expired/revoked refresh tokens every 24 hours.

**Frontend** — Vite 8, TanStack Router (file-based routes, `routeTree.gen.ts` auto-generated by router-plugin), TanStack Query + Form, Tailwind v4, shadcn/ui, ESLint 10.

`@/` path alias maps to `src/`. React-refresh rule allows `Route`, `AuthContext`, `ToastContext` exports — update `allowExportNames` in `eslint.config.js` if you add new named exports to those files.

## Auth

Dual delivery: same JWT works via **HttpOnly cookie** (web browsers) or **Authorization: Bearer** header (mobile/CLI/third-party). Cookie has priority — checked before header.

Login returns tokens in both cookie and JSON body. Refresh uses rotating tokens with theft detection: if a refresh token is used twice, ALL sessions for that user are revoked.

Rate limiting: Redis sliding window — 300 req/min global, 30 req/min on `/auth/*`. Trusts `CF-Connecting-IP` > `X-Real-IP` > `RemoteAddr`. Struct-based `RateLimiter` with `TxPipeline` for atomic Redis ops. Fail-open on Redis error.

## API

All responses wrapped in envelope: `{"success": true, "data": ...}` or `{"success": false, "error": {"code": "...", "message": "..."}}`.

Auto-generated route docs at `GET /docs/json` (via chi docgen). Only available when server is running.

## Frontend API Client

- `credentials: 'include'` — cookies sent on every request
- Auto-refresh on 401: silently calls `/auth/refresh`, retries once
- GET deduplication: concurrent identical GETs share one network call
- Mutation timeout: POST/PUT/DELETE abort after 30s

## State & Context

Auth and toast state live in `src/contexts/` (`auth-context.tsx`, `toast-context.tsx`). Hooks in `src/hooks/` (`use-auth.tsx`, `use-toast.tsx`) are thin context consumers. Provider/consumer separation prevents react-refresh warnings.

## Routes

SEO meta tags (title, description, Open Graph, Twitter Card) set per-route via TanStack Router `head()` API, plus static fallback in `index.html` (description, OG, Twitter, canonical, theme-color, noscript). `index.html` has full SEO meta including `<noscript>` branded fallback.

Static SEO files: `public/sitemap.xml`, `public/robots.txt`, `public/og-image.webp` (placeholder).

## Middleware

Ordering: RequestID → RealIP → CleanPath → Recover → Logger → SecurityHeaders → CORS → RateLimiter

- **Recover** — custom JSON panic output (`INTERNAL_ERROR`) instead of chi plain-text. Includes `request_id` in error logs.
- **Logger** — structured JSON via slog with `request_id`, `status`, `duration_ms`, `bytes_written`, optional `user_id`.
- **SecurityHeaders** — factory pattern with OWASP headers (HSTS 2y preload, CSP, X-Frame-Options DENY, etc.).
- **Validate** — returns `*T` instead of `(T, bool)`, uses `toSnakeCase` for field names in validation errors.
- **RateLimiter** — struct-based with `NewRateLimiter()` + `.Middleware()`. Uses `TxPipeline()` for atomic cleanup+count. Configurable `TrustedCIDRs`. Fail-open on Redis error.

## Database

SQL migrations in `backend/migrations/` via Bun migrator — auto-run on startup. Each migration has `.up.sql` + `.down.sql` pair, embedded via `//go:embed`. Tables: `users`, `refresh_tokens`.

## Pre-commit

Pre-commit hook via Husky + lint-staged at repo root. Runs on every `git commit`:
1. `go vet ./...` — backend code quality
2. `go build ./cmd/server` — backend compilation check
3. `lint-staged` → `eslint --fix` on staged `.ts/.tsx` files
4. `tsc --noEmit` — frontend type safety

Install automatically via `npm install` (prepare script triggers husky). Root `.gitignore` excludes `node_modules/`.

## Docker

**All stages use Docker Hardened Images (DHI)** from `dhi.io`:

| Service  | Build stage                  | Final stage                      |
| -------- | ---------------------------- | -------------------------------- |
| Backend  | `dhi.io/golang:1.26-debian13-dev` | `dhi.io/static:20250419-debian13`  |
| Frontend | `dhi.io/node:24-debian13-dev`     | `dhi.io/nginx:1.30-debian13`       |

Final images run as `nonroot`, have no package manager, and no shell (static) or minimal footprint (nginx). Healthchecks delegated to orchestration layer — Dockerfiles use `--mount=type=cache` and `COPY --link` for faster builds. Frontend uses `entrypoint.sh` for runtime `VITE_API_URL` injection.

## Notable

- No tests exist yet (no `_test.go` files). No Makefile. No `.env.example` (config in README).
- `backend/internal/sanitize/sanitize.go`: HTML-escapes user input (XSS prevention).
- JWT generation uses typed `accessTokenClaims` struct (not `MapClaims`) for compile-time safety.
- Go binary stripped with `-ldflags="-s -w"`. All Docker `COPY` instructions use `--link` for better cache reuse.
- `VITE_API_URL` env var (default `http://localhost:8080/api/v1`). `TOKEN_TYPE` env var (default `Bearer`).
- `.dockerignore` files in both `backend/` and `frontend/`.
- `KEYDB_ADDR` env var used in `main.go` (default `localhost:6379`). Not in Config struct — set via env directly.
- Logout uses `queryClient.invalidateQueries({ queryKey: ['auth', 'me'] })` for targeted cache invalidation.
- `AbortSignal.any()` in `api.ts` uses proper type guard instead of `as any` cast.
